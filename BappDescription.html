This extension provides coverage of all ten OWASP API Security Top 10 (2023) categories through active and passive scan checks. It reports findings labelled by OWASP API category so that scan output maps directly to the framework. Where checks overlap with native scanner coverage, issue details include references linking to the corresponding native issue definitions. The extension auto-detects the running edition and registers a UI tab only under the Professional edition.

Features

  • Full OWASP API Security Top 10 (2023) coverage across 15 checks, combining active probing and passive traffic analysis.
  • Active version probing on versioned API paths (e.g. /vN/) to detect deprecated API versions still reachable in production by probing /v(N-1)/, /v(N-2)/, and so on.
  • Content-compared broken object level authorization detection that fires only when a manipulated identifier returns a materially different response body, reducing noise from incidental 2xx responses.
  • HTTP Parameter Pollution detection that treats any response change from a duplicated parameter -- including a 200 → 400 flip -- as an override primitive worth reporting.
  • Optional AI integration that activates two behaviours when enabled: passive false-positive triage on noise-prone checks, and contextual discovery of domain-specific mass-assignment field candidates beyond the hardcoded defaults.
  • API-specific coverage for categories with no native equivalent: Unrestricted Resource Consumption, Unrestricted Access to Sensitive Business Flows, Improper Inventory Management, and Unsafe Consumption of APIs.
  • Each overlapping issue includes a reference line linking to the corresponding native issue definition, making it straightforward to cross-reference findings with native scanner results.

Usage

  1. Build the extension with mvn clean package -DskipTests to produce the JAR in the target/ directory.
  2. Load the JAR via Extensions → Installed → Add → Java.
  3. Confirm the extension loaded correctly by checking the Output tab for the startup banner, which shows the detected edition and whether AI features are active.
  4. Browse or proxy API traffic as normal. Passive checks run automatically against all traffic passing through the proxy.
  5. To trigger active checks, send API requests to the active scanner. All ten OWASP API categories will be probed, including version enumeration on any path matching the /vN/ pattern.
  6. Review findings in the Issues panel. Each issue is labelled with its OWASP API category. For checks that overlap native coverage, follow the Related Burp Scanner checks link in the issue detail to compare with native findings.
  7. If AI features are enabled, passive findings on noise-prone checks will be automatically triaged before reporting, and mass-assignment payloads will include domain-specific field suggestions derived from the observed request body.
